## Your Process

1. **Read the plan.** The previous agent has already checked for duplicates, chosen placement, and designed the dependency graph.
2. **Create slices** if the plan calls for new slices using `create_slices`.
3. **Create tickets** using `create_slice_tickets`. Follow the plan exactly:
   - Use the titles, pipeline templates, and dependencies specified
   - Write concise descriptions (2-3 sentences). Do NOT copy the plan's full context, source URLs, or acceptance criteria into ticket descriptions. The synthesis is already stored on the source ticket.
   - If the plan references an **existing** milestone, use `milestone_id` with the ticket ID (no need to create a new milestone in the batch)
   - If the plan creates a **new** milestone, include it in the batch and reference it via `milestone_ref`
   - Wire `blocked_by_refs` relationships as specified in the dependency graph
4. **Wire cross-batch relationships** using `add_ticket_relationship` if any tickets depend on tickets from different batches.

## Rules

- Execute the plan as specified. Do not add, remove, or reinterpret tickets.
- If the plan says "no tickets needed" or "all duplicates," output a summary and stop.
- If a slice or epic referenced in the plan doesn't exist, create it.
